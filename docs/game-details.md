# 各ゲーム詳細仕様

このドキュメントは、各ゲームを実装するための詳細仕様です。基本方針は「幹事が説明しなくても、画面の流れだけで遊べる」こととします。

## 1. 共通仕様

### 共通プレイモード

- 1台共有モード: 幹事または代表者のスマートフォンを回して遊ぶ
- 各自参加モード: QRコードまたはルームコードで参加者が自分の端末から参加する
- 初期実装では1台共有モードを優先する
- 各自参加モードは、参加者ルーム機能の追加後に対応する

### 共通画面

- ゲーム説明画面
- 参加者登録画面
- 設定画面
- プレイ画面
- 結果画面
- もう一度遊ぶ画面
- 全体初期化ボタン: 全ゲームの保存状態を消してトップ画面へ戻る

### 共通データ

```text
GameSession
- id
- gameType
- status
- players
- settings
- currentStep
- createdAt
- updatedAt
```

```text
Player
- id
- name
- displayOrder
- isActive
```

### 共通ステータス

- setup: 準備中
- playing: プレイ中
- result: 結果表示中
- finished: 終了

### 共通安全設計

- お題や質問はポジティブ、軽い自己開示、会話のきっかけになるものを優先する
- 容姿、年齢、収入、政治、宗教、病気、家族事情、恋愛経験、性的な話題は初期データに入れない
- 参加者が答えたくない場合にスキップできる導線を用意する
- 結果表示では順位や敗北を強調しすぎず、会話が続く文言にする

## 2. ジョハリの窓

### 体験ゴール

参加者が「自分ではそう思っていなかったけど、周りからはそう見えているんだ」と自然に話せる状態を作る。

### プレイ人数と時間

- 最小人数: 3人
- 推奨人数: 4人から8人
- 最大人数: 10人
- 1人あたりの所要時間: 3分から5分
- 全員で回す場合の目安: 15分から30分

### 画面遷移

1. ゲーム説明
2. 参加者登録
3. 特徴ワードセット選択
4. 対象者選択
5. 自己選択
6. 他者選択
7. 結果表示
8. 次の対象者選択
9. 全体終了

### 詳細ルール

- 対象者は自分に当てはまる特徴ワードを3個から7個選ぶ
- 他の参加者は対象者に当てはまる特徴ワードを3個から7個選ぶ
- 他者選択は合算して扱う
- 他者が1人でも選んだ特徴は「他者が選んだ」と判定する
- 「盲点の窓」には、対象者が選ばず他者が選んだ特徴を表示する
- 「未知の窓」は表示量が多くなりやすいため、初期表示では折りたたむ

### 設定項目

- 特徴ワードセット: ベーシック、仕事仲間向け、友人向け
- 自己選択数の下限と上限
- 他者選択数の下限と上限
- 他者選択を匿名にするか
- 全員分を順番に実施するか、1人だけ実施するか

### データ構造

```text
JohariWord
- id
- label
- category
- tone
```

```text
JohariAnswer
- targetPlayerId
- voterPlayerId
- selectedWordIds
- answerType: self | other
```

```text
JohariResult
- openAreaWordIds
- hiddenAreaWordIds
- blindAreaWordIds
- unknownAreaWordIds
```

### 判定ロジック

- 自分が選んだ、かつ他者も選んだ: 開放の窓
- 自分が選んだ、かつ他者が選ばなかった: 秘密の窓
- 自分が選ばなかった、かつ他者が選んだ: 盲点の窓
- 自分も他者も選ばなかった: 未知の窓

### 結果画面

- 4象限で表示する
- 各象限に短い説明を添える
- 盲点の窓はポジティブに見える文言にする
- 結果の保存、共有、画像化は後続機能とする

### 初期ワードセット

ベーシック:

- 話しやすい
- 聞き上手
- 明るい
- 落ち着いている
- 頼りになる
- 気配りができる
- 好奇心が強い
- 誠実
- 親しみやすい
- 発想が豊か
- 行動力がある
- 慎重
- ユーモアがある
- 面倒見がよい
- 粘り強い
- 素直
- 柔軟
- 集中力がある
- 場を和ませる
- 意外性がある

### MVP範囲

- 1台共有モード
- 参加者登録
- ベーシックワード固定
- 1人ずつ対象者を選んで実施
- 4象限結果表示

## 3. ウミガメのスープ

### 体験ゴール

参加者が短時間で推理を楽しみ、正解後に「なるほど」と会話が広がる状態を作る。

### プレイ人数と時間

- 最小人数: 2人
- 推奨人数: 3人から6人
- 最大人数: 10人
- 1問あたりの目安: 5分から12分

### 役割

- 出題者: 答えと解説を見て、質問に回答する
- 回答者: 質問をして真相を推理する
- 進行役: 出題者が兼任できる

### 画面遷移

1. ゲーム説明
2. 問題選択
3. 出題者確認
4. 問題文表示
5. 質問進行
6. ヒント表示
7. 回答確認
8. 解説表示
9. 次の問題へ

### 詳細ルール

- 回答者は「はい」「いいえ」で答えられる質問をする
- 出題者は必要に応じて「関係ありません」「どちらともいえません」を使える
- 質問が止まった場合、ヒントを1つずつ開ける
- 正解判定は出題者が行う
- 完全一致ではなく、真相の重要要素が出ていれば正解とする

### 設定項目

- 難易度: かんたん、普通、むずかしい
- 問題カテゴリ: 日常、勘違い、言葉遊び、行動の理由
- ヒント表示: あり、なし
- 質問履歴: 記録する、記録しない
- 制限時間: なし、5分、10分、15分

### データ構造

```text
LateralThinkingPuzzle
- id
- title
- difficulty
- category
- estimatedMinutes
- questionText
- answerSummary
- explanation
- hints
- tags
- safetyLevel
```

```text
QuestionLog
- id
- playerId
- questionText
- answerType: yes | no | irrelevant | unclear | note
- note
- createdAt
```

### 初期問題の条件

- 文章は80字以内を目安にする
- 真相は200字以内を目安にする
- 最低3つのヒントを持つ
- 飲み会で扱いやすい軽い内容にする

### 初期問題例

タイトル: 消えたケーキ

問題文:
机の上にあったケーキが消えた。誰も食べていないのに、ケーキはなくなっていた。なぜ？

真相:
ケーキは本物ではなく、誕生日用の飾りとして置かれた写真付きのカードだった。片付けのときにカードとして回収された。

ヒント:

- 食べ物としては扱われていない
- 誰かが持ち去った
- 見た目がケーキに見えた

### MVP範囲

- 問題一覧
- 問題文表示
- 出題者向け答え表示
- ヒント表示
- 解説表示
- 質問履歴は簡易メモ扱い

## 4. ワードウルフ

### 体験ゴール

お題を直接言えない緊張感の中で、会話と推理が自然に盛り上がる状態を作る。

### プレイ人数と時間

- 最小人数: 4人
- 推奨人数: 5人から8人
- 最大人数: 12人
- 1ゲームの目安: 5分から8分

### 画面遷移

1. ゲーム説明
2. 参加者登録
3. 設定
4. お題配布
5. 個別確認
6. 会話タイマー
7. 投票
8. 結果表示
9. 再戦

### 詳細ルール

- 多数派と少数派に似たお題を配る
- 自分のお題をそのまま言ってはいけない
- 会話終了後、少数派だと思う人に投票する
- 最多得票者が少数派なら多数派の勝ち
- 最多得票者が多数派なら少数派の勝ち
- 同票の場合、決選投票または少数派勝利を設定で選べる

### 設定項目

- 少数派人数: 1人、2人
- 会話時間: 3分、5分、7分
- 同票時ルール: 決選投票、少数派勝利
- お題カテゴリ: 全部、食べ物、日常、おでかけ、エンタメ、旅行、仕事仲間、人柄、恋バナ、夜の話題
- Hな話題: トグルスイッチでON/OFF
- Hな話題OFF: 大人向けカテゴリを表示せず、出題にも含めない
- Hな話題ON: 全部に大人向けも混ぜ、大人向けカテゴリだけでも遊べる
- お題の難しさ: やさしい、普通

### データ構造

```text
WordWolfTopicPair
- id
- majorityWord
- minorityWord
- category
- difficulty
- distanceLevel
```

```text
WordWolfAssignment
- playerId
- word
- role: majority | minority
```

```text
WordWolfVote
- voterPlayerId
- targetPlayerId
```

### お題設計基準

- 似ているが、経験や好みの話で差が出るものにする
- 説明しづらすぎる抽象語は避ける
- 片方だけ極端に有名、極端に狭いものは避ける
- 初対面でも話せる日常的な題材にする

### 初期お題セット

- ラーメン / うどん
- コーヒー / 紅茶
- 映画館 / カラオケ
- 海 / プール
- 焼肉 / しゃぶしゃぶ
- コンビニ / スーパー
- 花火 / イルミネーション
- 温泉 / サウナ
- 自転車 / 電車
- カレー / シチュー

### MVP範囲

- 参加者登録
- 少数派1人固定
- お題自動配布
- 1台共有で個別確認
- タイマー
- 投票と勝敗表示

## 5. NGワードゲーム

### 体験ゴール

普通の会話をしながら、言わせたい・言わないようにしたいという軽い駆け引きを楽しめる状態を作る。

### プレイ人数と時間

- 最小人数: 3人
- 推奨人数: 4人から8人
- 最大人数: 12人
- 1ゲームの目安: 5分から10分

### 画面遷移

1. ゲーム説明
2. 参加者登録
3. NGワード設定
4. 個別確認
5. 会話タイマー
6. アウト記録
7. 結果表示

### 詳細ルール

- 各参加者に1つずつNGワードを割り当てる
- 本人は自分のNGワードを知らない
- 他の参加者は、その人のNGワードを知っている
- 自分のNGワードを言ったら減点する
- 初期ルールでは脱落せず、最後まで参加できる
- 最も減点が少ない人を勝者とする

### 設定項目

- 制限時間: 5分、10分、15分
- 減点方式: 1回ごとに減点、初回だけ減点
- NGワード数: 1個、2個
- ワード難度: かんたん、普通
- 手動編集: あり、なし

### データ構造

```text
NgWordAssignment
- playerId
- word
- penaltyCount
```

```text
NgWordPenalty
- id
- playerId
- word
- recordedByPlayerId
- createdAt
```

### NGワード設計基準

- 日常会話で自然に出やすい言葉にする
- 露骨に言わせにくい言葉は避ける
- 悪口や相手を困らせる言葉は入れない
- 場の話題が広がる単語を優先する

### 初期NGワードセット

かんたん:

- すごい
- なるほど
- たしかに
- やばい
- おいしい
- いいね
- ほんと
- なんで
- ありがとう
- ちょっと

普通:

- 仕事
- 明日
- 最近
- 好き
- 眠い
- 忙しい
- 旅行
- ごはん
- 休み
- 楽しい

### MVP範囲

- 参加者登録
- NGワード自動割り当て
- 1人1ワード
- 減点記録
- 結果表示

## 6. 二択トーク

### 体験ゴール

すぐ答えられる二択から、理由やエピソードを話しやすくする。

### プレイ人数と時間

- 最小人数: 2人
- 推奨人数: 3人から12人
- 最大人数: 20人
- 1問あたりの目安: 1分から3分

### 画面遷移

1. ゲーム説明
2. カテゴリ選択
3. お題表示
4. A/B投票
5. 結果表示
6. コメントタイム
7. 次のお題

### 詳細ルール

- 参加者はAまたはBを選ぶ
- 回答後に全体結果を表示する
- 少数派、またはランダムに選ばれた人から理由を聞く
- 回答したくない人はスキップできる
- お題は正解のない価値観・好み・日常の選択にする

### 設定項目

- カテゴリ: 全部、アイスブレイク、食べ物、休日、暮らし、会話、エンタメ、おでかけ、仕事仲間、もしも、日常
- Hな話題: トグルスイッチでON/OFF
- Hな話題OFF: 大人向けカテゴリを表示せず、出題にも含めない
- Hな話題ON: 全部に大人向けも混ぜ、大人向けカテゴリだけでも遊べる
- 表示順: ランダム、カテゴリ順
- コメント指名: 少数派、ランダム、指名なし
- 投票方法: 人数だけ、参加者名あり

### データ構造

```text
TwoChoicePrompt
- id
- category
- question
- optionA
- optionB
- tone
```

```text
TwoChoiceVote
- promptId
- playerId
- choice: A | B | skip
```

### 初期お題セット

アイスブレイク:

- 旅行するなら 海 / 山
- 朝型 / 夜型
- 甘いもの / しょっぱいもの
- 計画派 / 直感派
- 家でゆっくり / 外に出かける

食べ物:

- ラーメン / カレー
- 焼肉 / 寿司
- コーヒー / 紅茶
- 朝ごはんは ごはん / パン
- デザートは アイス / ケーキ

休日:

- 予定を入れる / 何も決めない
- 遠出 / 近場
- 映画 / 散歩
- ひとり時間 / 誰かと会う
- 早起き / 寝坊

### MVP範囲

- お題ランダム表示
- A/B投票
- 結果の人数表示
- 次のお題への遷移

## 7. 第一印象ランキング

### 体験ゴール

ポジティブな第一印象を共有し、選ばれた人が少し嬉しくなる状態を作る。

### プレイ人数と時間

- 最小人数: 4人
- 推奨人数: 5人から10人
- 最大人数: 12人
- 1問あたりの目安: 2分から4分

### 画面遷移

1. ゲーム説明
2. 参加者登録
3. お題カテゴリ選択
4. お題表示
5. 投票
6. 結果表示
7. コメントタイム
8. 次のお題

### 詳細ルール

- お題に一番当てはまりそうな人を1人選ぶ
- 自分に投票できるかどうかは設定で切り替える
- 最多得票者を表示する
- 同票の場合は複数人を表示する
- 選ばれた人は一言コメントしてもよい
- コメントは任意にする

### 設定項目

- カテゴリ: やさしい、盛り上がる、仕事仲間向け、友人向け
- 自分への投票: あり、なし
- 結果表示: 1位だけ、全体票数
- 連続選出制限: あり、なし

### データ構造

```text
ImpressionPrompt
- id
- category
- text
- tone
```

```text
ImpressionVote
- promptId
- voterPlayerId
- targetPlayerId
```

### お題設計基準

- 選ばれた人が困らない表現にする
- 能力、雰囲気、行動イメージなどポジティブな観点にする
- からかい、容姿評価、私生活の詮索につながるものは避ける

### 初期お題セット

やさしい:

- 一番聞き上手そうな人
- 一番おいしいお店を知っていそうな人
- 一番旅行の計画がうまそうな人
- 一番朝の準備が早そうな人
- 一番道に迷わなそうな人

盛り上がる:

- 一番サプライズが上手そうな人
- 一番フットワークが軽そうな人
- 一番カラオケの選曲がうまそうな人
- 一番写真を撮るのがうまそうな人
- 一番イベントを企画するのがうまそうな人

### MVP範囲

- 参加者登録
- お題ランダム表示
- 1人に投票
- 最多得票者表示
- 同票表示

## 8. 匿名質問箱

### 体験ゴール

匿名だからこそ出しやすい軽い質問で、会話のきっかけを増やす。

### プレイ人数と時間

- 最小人数: 3人
- 推奨人数: 4人から12人
- 最大人数: 20人
- 1問あたりの目安: 1分から3分

### 画面遷移

1. ゲーム説明
2. 注意事項表示
3. 質問投稿
4. 質問一覧
5. 質問表示
6. 回答
7. 回答済みにする
8. 次の質問

### 詳細ルール

- 参加者は匿名で質問を投稿する
- 投稿された質問からランダムまたは順番に表示する
- 回答者は全員、指名、希望者のいずれかにする
- 答えにくい質問はスキップできる
- 進行役は質問を非表示または削除できる

### 設定項目

- 表示順: ランダム、投稿順
- 回答方式: 全員、指名、希望者
- 質問投稿: 自由入力、テンプレート選択
- 進行役確認: あり、なし
- スキップ: あり

### データ構造

```text
AnonymousQuestion
- id
- text
- category
- status: pending | answered | skipped | hidden
- createdAt
```

```text
QuestionBoxRound
- currentQuestionId
- displayMode
- responderPlayerId
```

### 投稿前注意文

```text
相手が困る質問、個人情報に踏み込む質問、答えを強制する質問は避けましょう。
答えにくい質問はスキップできます。
```

### 初期テンプレート質問

- 最近ちょっと嬉しかったことは？
- 子どものころ好きだった遊びは？
- 今年やってみたいことは？
- 旅行で行ってよかった場所は？
- 実は少し得意なことは？
- 最近おすすめしたいものは？
- もし1日休みが増えたら何をする？
- 最近笑ったことは？
- 今年食べておいしかったものは？
- 小さなこだわりは？

### MVP範囲

- 質問投稿
- 質問一覧
- ランダム表示
- 回答済み管理
- スキップ

## 9. 初期実装の推奨順

1. トップ画面
2. 二択トーク
3. ワードウルフ
4. NGワードゲーム
5. ウミガメのスープ
6. 第一印象ランキング
7. ジョハリの窓
8. 匿名質問箱

理由:

- 二択トークはデータ構造と画面遷移が最も単純で、アプリ全体の操作感を作りやすい
- ワードウルフとNGワードゲームは参加者登録、個別確認、投票、結果表示の共通部品を育てやすい
- ウミガメのスープは問題データの品質が重要なため、初期コンテンツ整備と並行して作る
- ジョハリの窓と匿名質問箱は心理的安全性の設計が重要なため、UI文言を丁寧に詰めてから実装する
