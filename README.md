# 飲み会用アプリ

飲み会、懇親会、チーム交流で使えるミニゲーム集です。  
幹事のスマホやPCから開いて、その場で参加者名を入れればすぐ遊べます。

## 公開URL

静的版 v1 は GitHub Pages で公開しています。

[https://ysky24-cell.github.io/nomikai-app/](https://ysky24-cell.github.io/nomikai-app/)

## 遊び方

1. 公開URLを開きます。
2. トップ画面から遊びたいゲームを選びます。
3. 参加者名を登録します。
4. 必要に応じて、設問数、カテゴリ、Hな話題のON/OFFなどを選びます。
5. 画面の進行説明に沿ってゲームを進めます。
6. 飽きたときや最初からやり直したいときは「初期化」でトップへ戻れます。

ゲームごとに、進め方、判定方法、盛り上げ方、注意事項を画面内に表示します。

## 静的版 v1 の特徴

- GitHub Pages だけで公開できる静的アプリです。
- データベース、ログイン、サーバー同期は使いません。
- 参加者、投票、進行状況はブラウザ内の状態と `localStorage` に保存します。
- 多くのゲームで、5 / 10 / 20 / 30 / 50 / 100 / 300 問から設問数を選べます。
- 多くのゲームで、Hな話題をトグルスイッチでON/OFFできます。
- 人狼ゲームにはHな話題や設問数選択はありません。
- 画像付きカードで各ゲームを選べます。
- 全体初期化で、全ゲームの状態を消してトップ画面へ戻れます。

## 代表ゲームの補足

### 人狼ゲーム

- 6〜12人の参加者に、人狼、村人、占い師、騎士、霊媒師を自動配役します。
- スマホを順番に回し、本人だけが役職を確認できます。
- 夜の襲撃、占い、護衛、霊媒確認を司会用画面で順番に記録できます。
- 昼の議論タイマー、順番投票、同票時の再投票、勝敗判定まで画面内で進行できます。
- Hな話題や大人向けお題はありません。

### ワードウルフ

- 300ペアのお題プールを搭載しています。
- 進行役が迷わないよう、開始前に進め方、会話ルール、投票ルールを表示します。
- Hな話題はトグルスイッチでON/OFFできます。

### 二択トーク

- 通常300問、大人向け30問の設問プールを搭載しています。
- A/Bで選び、理由を話して会話を広げます。
- Hな話題はトグルスイッチでON/OFFできます。

### ウミガメのスープ

- 300問の短問プールを搭載しています。
- 出題者だけが真相を表示でき、ヒントは段階的に開けます。
- 質問の返し方、詰まった時の進め方を画面内に表示します。

### ジョハリの窓

- 300語の特徴ワードを搭載しています。
- 本人の選択、他者の選択を分けて進めます。
- 結果を「開放」「盲点」「秘密」「未知」の4つの窓で表示します。

## 既知の制限

- 静的版では、複数人のスマホ間でリアルタイム同期はできません。
- Dockerルーム版では、全ゲーム共通の開始、進行、完了、待機戻しを同期できます。
- Dockerルーム版の人狼ゲームは、配役、夜行動、サーバー時刻基準の昼議論タイマー、投票、勝敗判定のルーム同期に対応しています。
- Dockerルーム版のワードウルフは、各自のお題確認、サーバー時刻基準の会話タイマー、順番投票、結果表示のルーム同期に対応しています。
- Dockerルーム版の二択トークは、各自投票、集計、結果表示のルーム同期に対応しています。
- Dockerルーム版のマジョリティゲーム / 大人数マジョリティは、各自投票と集計表示のルーム同期に対応しています。
- Dockerルーム版の第一印象ランキングは、ルーム参加者取り込み、各自投票、集計、結果表示のルーム同期に対応しています。
- Dockerルーム版の匿名質問箱は、ルーム参加者取り込み、匿名質問投稿、質問開封、スキップ、完了のルーム同期に対応しています。
- Dockerルーム版のNGワードゲームは、ルーム参加者取り込み、本人だけ非表示のNGワード配布、踏んだ記録、サーバー時刻基準のタイマー、結果表示のルーム同期に対応しています。
- Dockerルーム版の人狼の役職、ワードウルフのお題、NGワードは、ルーム取得とSocket.IO配信時に参加者ごとに見える内容を分けています。
- Dockerルーム版のジョハリの窓は、ルーム参加者取り込み、本人選択、周りの選択、4つの窓の結果表示のルーム同期に対応しています。
- Dockerルーム版のウミガメのスープは、ルーム参加者取り込み、質問ログ、ヒント、出題者だけの真相確認、全員への真相表示、次の問題への進行のルーム同期に対応しています。
- Dockerルーム版の山手線ゲームは、ルーム参加者取り込み、手番、回答記録、パス、アウト記録、次のお題への進行のルーム同期に対応しています。
- Dockerルーム版の逆さ言葉ゲーム / 外来語禁止ゲームは、ルーム参加者取り込み、手番、セーフ、パス、アウト記録、次のお題への進行のルーム同期に対応しています。
- Dockerルーム版の曲名連想クイズ / お絵描きクイズ / 記憶だけでロゴを書く / 変な一言カルタ / エモヒント連想 / 人物当てヒントクイズ / 鼻歌イントロドンは、ルーム参加者取り込み、親、各自回答、正解表示、得点のルーム同期に対応しています。
- Dockerルーム版の2つの真実と1つの嘘 / 価値観メーター / スマホ早打ちゲームは、ルーム参加者取り込み、各自回答、集計、結果表示、得点のルーム同期に対応しています。
- Dockerルーム版のひとこと演技ゲームは、ルーム参加者取り込み、演者、感情選択、各自回答、結果表示、得点のルーム同期に対応しています。
- Dockerルーム版のカウントアップゲーム / ドキドキはずれカード / 安全はずれ抽選 / 腕相撲トーナメントは、ルーム参加者取り込み、手番、数字、カード、対戦結果のルーム同期に対応しています。
- Dockerルーム版の飲み会すごろく / 人生イベントすごろく / シンプル陣取り / 資源交渉トークは、ルーム参加者取り込み、手番、コマ位置、盤面、資源のルーム同期に対応しています。
- Dockerルーム版の定番ゲームパックは、ルーム参加者取り込み、パック内ミニゲーム選択、手番、回答、投票、結果表示、得点、次のお題への進行のルーム同期に対応しています。
- Dockerルーム版でも、上記以外の補助表示、ローカルUI状態、画像アップロード、音源再生、長期履歴保存まではまだ完全同期していません。
- Dockerルーム版の役職やお題は参加者別にマスクして配信しますが、DB内部のルーム状態には保存されます。サーバー運用者からも見えない形の暗号化は未対応です。
- Dockerルーム版のゲーム内フィールド単位の権限検証は段階的に強化中です。同期対象外の細かい補助操作は、信頼できる参加者同士で遊ぶ前提です。
- ブラウザを変えると保存状態は引き継がれません。
- `localStorage` を消す、または全体初期化を押すと進行中の状態は消えます。
- GitHub Pages版では、サーバー処理、DB保存、WebSocket通信は使いません。
- 音源、画像アップロード、ログイン、履歴保存は静的版では扱いません。
- 腕相撲など身体を使うゲームは、画面内の安全注意を読んだうえで無理なく遊ぶ想定です。

## Docker / ルーム版について

静的版は `main` ブランチで維持します。  
Docker、DB、WebSocketを使う多人数ルーム参加版は、別ブランチで進めます。

想定構成:

- `web`: Reactフロント
- `api`: Node.js + Express + Socket.IO
- `db`: PostgreSQL
- `redis`: ルーム状態とリアルタイム同期補助

実装済み:

- ルーム作成
- ルームコードで参加
- ニックネーム登録
- ホストによるゲーム開始と進行
- ホスト交代
- ホストによる参加者削除
- ホスト操作APIとSocket.IO状態更新の基本的なサーバー側権限チェック
- 人狼、ワードウルフ、NGワード、二択トーク、第一印象ランキング、マジョリティ系、匿名質問箱、ウミガメのスープ、ジョハリの窓、山手線ゲーム、定番ゲームパック、当てる系、回答・集計系、演技系、手番・抽選・対戦系、盤面系の本人欄・投票・記録・手番・盤面単位のサーバー側検証
- 人狼の役職、ワードウルフのお題、NGワードの参加者別マスク配信
- 保存済みルームへの再接続と役割更新
- 参加者画面へのリアルタイム同期
- 全ゲーム共通の「この端末で開く」接続
- 人狼ゲームの配役、夜行動、サーバー時刻基準の昼議論タイマー、投票、勝敗判定のルーム同期
- ワードウルフの各自お題確認、サーバー時刻基準の会話タイマー、順番投票、結果表示のルーム同期
- 二択トークの各自投票、集計、結果表示のルーム同期
- マジョリティゲーム / 大人数マジョリティの各自投票、集計表示のルーム同期
- 第一印象ランキングのルーム参加者取り込み、各自投票、集計、結果表示のルーム同期
- 匿名質問箱のルーム参加者取り込み、匿名質問投稿、質問開封、スキップ、完了のルーム同期
- NGワードゲームのルーム参加者取り込み、本人だけ非表示のNGワード配布、踏んだ記録、サーバー時刻基準のタイマー、結果表示のルーム同期
- ウミガメのスープのルーム参加者取り込み、質問ログ、ヒント、出題者だけの真相確認、全員への真相表示、次の問題への進行のルーム同期
- ジョハリの窓、山手線ゲーム、逆さ言葉ゲーム、外来語禁止ゲーム、当てる系ゲーム、盤面系ゲームのルーム同期
- 2つの真実と1つの嘘、価値観メーター、スマホ早打ちゲームの回答、集計、結果表示のルーム同期
- ひとこと演技ゲームの演者、感情選択、各自回答、結果表示、得点のルーム同期
- カウントアップゲーム、ドキドキはずれカード、安全はずれ抽選、腕相撲トーナメントの手番、数字、カード、対戦結果のルーム同期
- 定番ゲームパックのルーム参加者取り込み、パック内ミニゲーム選択、手番、回答、投票、結果表示、得点、次のお題への進行のルーム同期

未対応:

- 上記以外の補助表示、ローカルUI状態、画像アップロード、音源再生、履歴保存の完全同期
- 観戦者専用画面
- DB内部の役職やお題をサーバー運用者にも読めない形にする暗号化
- 同期対象外の細かい補助操作に対するサーバー権限検証
- 本番AWS用のHTTPS、ドメイン、認証、監視

ローカルでは次のコマンドで、Reactフロント、API、PostgreSQL、Redisをまとめて起動できます。

```bash
docker compose -p nomikai-app up --build
```

起動後のURL:

- フロント: [http://localhost:5173/nomikai-app/](http://localhost:5173/nomikai-app/)
- APIヘルスチェック: [http://localhost:3000/health](http://localhost:3000/health)

詳しくは [Docker / ルーム版セットアップ](docs/docker-room-setup.md) を参照してください。
